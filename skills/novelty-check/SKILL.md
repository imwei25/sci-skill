# 选题前置把关：新颖性裁定 + 预注册锁

`topic-selection` 负责**发散**（提候选、多棱镜打分）；本技能负责在动手前**收敛与锁定**两件贵而易错的事：
1. **新颖性裁定**——把"我觉得这是空白"换成"对着真实文献和注册库查过、给出可辩护的裁定"。最贵的实验是那个不该做的实验。
2. **预注册锁**——在看结局数据之前把假设与主分析计划冻结，事后偏离要记录披露。这是防 **HARKing**（Hypothesizing After Results are Known，看到结果再编个假设去套）和 p-hacking 的结构性护栏。

> **绝不虚构**："尚无人研究""已注册""某试验在做"这类论断必须来自真实检索——查不到就说"未检索到，需进一步确认"，不臆断。

## 何时用、何时跳过
- **用**：topic-selection 选出 1–2 个主攻方向后，正式写标书/开题/建库前；或用户直接问"这题新不新 / 帮我做研究注册"。
- **跳过**：还在发散提候选（用 `topic-selection`）；只想摸领域背景（用 `research-scan`）；纯回顾性、数据早已存在且不涉及预设假设检验的探索（新颖性仍建议查，预注册可略）。

## 一、新颖性裁定

### 四种裁定
| 裁定 | 含义 | 决定 |
|---|---|---|
| **真新颖** | 没有已发表答案，也没有在做的试验/系统综述覆盖 | 进入研究设计 |
| **增量** | 已被回答但只在更窄人群/更旧数据/更低证据级别；你的研究能加**有意义的精度或人群外推** | 可做，但**差异点必须在引言和预注册里写明** |
| **已被回答** | 有充分把握度的 RCT、系统综述或现行指南已定论 | **杀或重构**——说清现有证据为何不能回答你这个具体版本 |
| **有争议** | 存在互相冲突的高质量证据、领域无共识 | 强正当性——设计成**去裁决分歧**，而不是"再投一票" |

### 检索扫描（用我们套件的管道，别用记忆）
> ⚠️ 本环境 **NCBI/PubMed 常被阻断**，所以**主力用国内可达的 Europe PMC**，境外源作增强。详细步骤、查询构造、诚实性 10 项清单与产出模板见 **[references/novelty-sweep.md](references/novelty-sweep.md)**。要点：
- **主库检索**：`literature-review` 的 `search.py`（Europe PMC，国内可达，含 `citedByCount` 可按高被引排序）；有 MCP/境外网络时 `search-lit` 增强。关键文献读全文用 `fulltext-retrieval`。
- **系统综述 / 协议注册库**（防你的研究还没做就已冗余）：Cochrane Library、**PROSPERO**（在做的 SR 协议）、Epistemonikos。
- **试验注册库**（有在做的 RCT 会 2–3 年后回答同问题）：**ChiCTR（中国，首选）**、ClinicalTrials.gov、ISRCTN。
- **广覆盖增强**：OpenAlex（按被引排序找最有影响力的直接答案；⚠️ 2026-02 起需 API key，没 key 就退回 Europe PMC 人工归纳并标"仅线索"）、Semantic Scholar（influentialCitationCount）。
- **中文临床题**再查 CNKI / 万方 / SinoMed（无免费 API，用 WebFetch 或让用户代查），别漏中文文献里的既有工作。
- **邻近领域/同义词**：至少 2 组同义词、跑一次邻近领域版本（如儿科题查成人版是否已有答案）。

### 裁定规矩
- 给"真新颖"之前，**[references/novelty-sweep.md](references/novelty-sweep.md) 里的诚实性清单 10 项必须全勾**，缺一项就补检索再判。
- 关键论断至少 2 个独立来源；**存在性核验**交 `reference-check`，**"是否真支持"由主代理回读摘要确认**（别脚本一跑完就说"已核实"）。
- 最近的 5–10 篇最接近工作要人读、由用户（💡）拍最终板；AI 负责跑扫描、给证据。
- 产出《新颖性裁定记录》到工作区（模板见 references），含查询串、注册库结果、最接近的文献表、差异点陈述、裁定与决定。

## 二、预注册锁（写结局数据之前做）

**为什么**：先看数据再决定"检验什么、怎么分组、报哪个终点"，几乎必然高估效应、制造假阳性。预注册把这一步前移并冻结，注册后再改就是可见的偏离而非隐形的自由度。

**冻结哪些**（缺的向用户要真实信息，不替编）：
- **主假设**（可证伪的陈述）与 **PICO**。
- **主要结局**（单一、预先定义、含测量时点与方法）；次要结局单列。
- **主分析方法**：主要结局用什么检验/模型、协变量、亚组与交互（预先声明的才算验证性，其余一律标探索性）、多重比较处理、把握度/样本量依据（用 `clinical-stats` 先验估算，**不做事后 power**）。
- **纳入/排除标准、暴露/干预与对照定义、缺失数据处理计划**。

**设计层统计护栏（冻结主分析时逐条对照，拦截致命设计错误）**：
- **单一主要结局**：只设一个主要结局；确需共同主要结局(co-primary)则**必须做多重性校正(Bonferroni 等)并按联合把握度重算样本量**，别用多个终点"择优报"。
- **DAG 三律**（选协变量前先画因果图）：**调整混杂因子**；**不调整中介**（会吸掉真实效应）；**绝不调整对撞因子(collider)**（会凭空造出假关联）。别"把能测的都塞进模型"。
- **回归事件数护栏 EPV≥10**：logistic/Cox 每个自变量至少 ~10 个结局事件；不够就减少变量、用惩罚回归(Firth)或改线性入模，别硬塞一堆协变量导致过拟合。
- **样本量两种范式别混**：**差异型**（检验有无差异，给差值/SD/率/HR，用 `clinical-stats`）vs **精度型**（估计一个量的精度，按目标 95%CI 宽度/Wilson 区间算）——按研究目的选对，观察性/患病率研究常是精度型。
- **缺失数据计划**按比例分档（<5% 完整病例 / 5–40% 多重插补 m≥20 声明 MAR / >40% 存疑），LOCF 禁用——细则见 `data-analysis` 的[统计报告清单](../data-analysis/references/stat-reporting-checklist.md)。

**去哪注册**（据研究类型选，注册号是真实产物，别编）：
| 类型 | 平台 |
|---|---|
| 临床试验（干预性） | **ChiCTR**（中国）/ ClinicalTrials.gov —— **多数期刊要求入组前注册** |
| 系统综述 / Meta | **PROSPERO**（筛文献前注册协议） |
| 一般研究 / 观察性 / 假设 | **OSF**（osf.io，可注册假设与分析计划） |

**产出**：预注册草案写到工作区（模板见 [references/preregistration.md](references/preregistration.md)），提示用户去对应平台正式提交、回填真实注册号。**注册前后设计如有变更，记入"偏离日志"**，投稿时如实披露。

## 衔接
- 上游：`topic-selection`（拿到选定候选与其引用）、`research-scan`（邻近工作地图）。
- 下游：裁定为**真新颖/增量/有争议**且预注册锁定后 → `grant-proposal`（写标书）或 `write-paper`（Methods 直接引用预注册的主假设与分析计划，保持一致）。深挖 prior-art 用 `deep-research`。

## 产出与交付
- 《新颖性裁定记录》→ 工作区 `outputs/novelty_assessment.md`；预注册草案 → `outputs/preregistration.md`。
- 出 PDF 交 `render-pdf-doc`（中文报告指定 `--cjk-font`：本地 `Microsoft YaHei`，服务器 `Noto Sans CJK SC`）。

## 硬约束
- 不虚构文献、注册号、"已注册/在做"状态、"尚无人研究"结论——查不到就如实说"未检索到，需进一步确认"。
- 新颖性裁定要可辩护、有据；**"真新颖"必须过完 10 项诚实性清单**。
- 预注册的假设/结局/分析计划一经用于对外注册，改动即为可见偏离，须记录披露；**不得用观察到的结果反向修改主假设**（HARKing）。
