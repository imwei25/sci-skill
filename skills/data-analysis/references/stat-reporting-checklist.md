# 统计报告清单（交付统计结果 / 写进论文前**逐条过一遍**）

> 审稿人挑统计毛病，一半不是"算错"，而是"报得不专业/不完整"。本清单把顶刊统计报告规范固化成硬条目——**每次产出统计结论、写 Methods/Results 前必过**。做不到的项如实标注，别静默省略。

## 一、每个结果的四要素（缺一不可）
每个比较/关联都要给：**① 检验名 ② 每组样本量 n ③ 效应量 ④ 95% 置信区间**。
- ❌ 只写 "P<0.05" 或 "差异有统计学意义" —— 不合格。
- ✅ "治疗组较对照组收缩压平均低 6.2 mmHg（95%CI 2.1–10.3，Welch t，n=60 vs 58，P=0.003）"。
- **95%CI 优先于单独 p 值**：CI 同时给出方向、幅度与精度；p 只说"是否为偶然"。

## 二、数字格式（照此写，别自由发挥）
| 量 | 规则 | 例 |
|---|---|---|
| p 值 | 2–3 位有效数字；<0.001 就写 `P<0.001`（别写 P=0.000） | P=0.034、P=0.007、P<0.001 |
| 百分比 | 保留 1 位小数，并给分子/分母 | 34.2%（68/199） |
| OR / RR / HR | 保留 2 位小数 + 95%CI | OR 1.71（95%CI 0.83–3.54） |
| 均值±SD / 中位数[IQR] | 与原始数据精度一致，别过度精确 | 59.6±7.0；4.5[2.2, 8.1] |
| 相关系数 r / β | 2 位小数 + 95%CI（能给就给） | r=0.42（95%CI 0.28–0.54） |

## 三、按研究类型的必报项
- **RCT**：CONSORT 流程图；基线表 Table 1 **不放组间 p 值**（随机化后组间差异按定义是偶然，报 p 是概念错误）；主要结局给效应量 + 95%CI；有临床意义时给 **NNT**。
- **观察性研究**：**同时报未校正与校正估计**，并**明确列出校正了哪些混杂**；说明混杂选择依据（DAG / 先验 / 变量筛选）。
- **诊断试验**：先给 **TP/FP/FN/TN 的 2×2 原始四格表**（可复核一切指标的基础），再报敏感度、特异度、PPV、NPV、LR、**ROC-AUC**，**每个都带 95%CI**；别只报"准确率"。对齐 STARD 2015。
- **生存分析**：KM 曲线 + log-rank；多因素 Cox 报 HR+95%CI，并**检验 PH 假设**（Schoenfeld 残差），违背要说明处理（分层/时依协变量）。

## 四、多重比较与探索性
- 一次比多对/多终点 → **必须校正**：Bonferroni（保守）或 Benjamini–Hochberg FDR（多终点推荐）。报校正方法与校正后 p。
- **探索性 / 亚组分析明确标注"探索性"**，不得当主要结论呈现；亚组效应要报**交互作用检验**，别只做组内分别检验。

## 五、缺失数据（按缺失比例选方法，别默默丢）
- **必报每个关键变量的缺失百分比**，并说明机制假设（MCAR/MAR/MNAR）。
- 按缺失比例的处理决策：

| 缺失比例 | 机制 | 处理 |
|---|---|---|
| <5% | 近似 MCAR | 可完整病例分析（complete-case），说明即可 |
| 5–40% | MAR | **多重插补(MI)**，报**插补份数 m≥20**、插补模型与纳入变量，并**声明 MAR 假设** |
| >40% | 任意 | 数据质量存疑，慎用该变量或如实交代重大局限 |

- ⚠️ **LOCF（末次观测结转）当代指南不推荐、基本禁用**——它系统性低估变异、易造偏倚；纵向缺失用 MI 或混合效应模型。
- **任何插补/删补都要做敏感性分析**（对比不同处理下主结论是否稳健），别只报一种。
- 别默默做完整病例分析而不交代——那会引入选择偏倚。

## 六、可复现三件套（Methods 里写清）
- **软件及版本**（如 Python 3.11 / scipy 1.13，或 R 4.3 + 包名版本）。
- **显著性水平与单双侧**（通常 α=0.05，双侧）。
- 每个"结果"里出现的分析，Methods 里都要有对应描述（**方法↔结果一一对应**）。

## 七、绝对红线
- ⚠️ **不做事后功效分析（post-hoc power）**：已完成研究报效应量+95%CI，别用观察到的效应倒推 power（方法学上无意义甚至误导）。设计阶段的先验样本量估算用 `clinical-stats`。
- **三处数字一致**：同一个数在摘要、正文、表、图里必须完全相同。
- 每个数字都能回溯到统计脚本的原始输出行——**找不到出处的数字一律删或复核**，绝不"看着对就放过"。
