# 对抗红队评审（投稿前的"科学有效性闸门"）

> 常规逐节评审查的是"写得好不好、规范全不全"；**对抗红队查的是另一件事：这个核心结论会不会本身就是偶然/混杂/偏倚/p-hacking 造出来的假象**——即使它通过了所有格式与规范检查。二者互补，别相互替代。
>
> **什么时候跑**：作者投稿前自查的最后一道闸（对应 `write-paper` 收尾、`sci-pilot` 定稿前）；或用户明确要"往死里挑这篇的结论站不站得住"。成本高于常规评审，按需跑。

## 做法：四个"单一视角"的敌意审稿人
每个审稿人**只从一个角度进攻、不做平衡**（平衡是主代理最后综合的事）。支持子代理的框架可四个并行、各自独立取证；不支持则主代理逐个视角走一遍。

**1. 统计审稿人**
- 主要分析是否与预先计划一致？有没有"多重路径择优"（garden of forking paths）——试了很多分析只报显著的？
- 多重比较校正了吗？把握度够吗？检验的前提假设（正态/方差齐/独立/PH）验过吗？
- 每个 p 值是否都配了效应量 + 95%CI？"显著"是不是只靠一个勉强 <0.05 的 p 撑着？
- 存疑的统计可交 `data-analysis` 复算。

**2. 方法学审稿人**
- 研究设计能不能回答这个问题？**因果措辞**（"导致""改善""保护"）有没有超出设计（观察性研究却下因果结论）？
- 混杂控制够吗（画得出 DAG 吗）？选择偏倚、信息/测量偏倚、失访/缺失偏倚？
- 外推边界诚实吗——单中心/特定人群的结论有没有被说成普适？

**3. 新颖性审稿人**
- 这个"发现"是不是已经有人做过（或只是换个人群的重复）？新颖性论证是**研究前**就做的，还是事后补的？
- 有没有"新颖性注水"（把增量说成突破）、"阴性结果化妆"（无效说成"提示趋势"）？
- 对照 `novelty-check` 的新颖性裁定记录（若有）再攻一遍。

**4. 诚信 / 临床审稿人**
- 摘要里**每个数字**都能回溯到原始数据或统计输出吗（随手抽查几个）？
- 参考文献都能解析到真实 PMID/DOI 且**内容与转述吻合**吗（可疑的交 `reference-check`，转述准确性要回读摘要）？
- 伦理/注册/知情同意披露齐吗？如果这个结论是假的、被临床采信，会造成多大伤害？

## 关键一步：Steelman the Null（给"无效假设"做最强辩护）
四审之后，专门再做一遍**反方最强论证**：竭尽全力论证"这个发现完全可以由**偶然 / 混杂 / 测量偏倚 / 选择偏倚 / p-hacking** 解释，并不真实"。
- 如果论文**扛不住**这个最强反驳 → 讨论必须补强对应的排除论证，或把结论**降级为探索性**。
- 这是整个对抗评审里最值钱的一招：它逼作者证明"我的解释比'其实是假象'更可信"，而不是默认自己对。

## 判决（主代理综合四审 + steelman 后给）
| 判决 | 含义 | 行动 |
|---|---|---|
| **KILL** | 核心发现可能是假象，现有证据支撑不住 | 退回选题/设计（`novelty-check`）或大改后重跑本评审 |
| **MAJOR** | 有实质缺陷但可修 | 实质性修改后再跑一遍对抗评审 |
| **MINOR** | 只剩"应改"的小问题 | 改完即可进入投稿准备 |
| **PASS** | 核心结论扛住了对抗 | 进入投稿准备 |

## 两条规矩
- **"KILL 不是失败。"** 投稿前自己 KILL 掉一个站不住的结论，远比发表后被撤稿便宜。早杀省钱。
- **人类拍板 KILL/PASS**：AI 红队只负责给出最硬的攻击与证据，最终"这结论要不要放弃"由作者（💡人类）决定，不代劳。
