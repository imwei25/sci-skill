# 论文 / 标书评审技能

以严谨、建设性的审稿人视角给出评审意见。参考 K-Dense / davila7 的 peer-review 技能。**目标是帮作者改好，不是挑刺羞辱。**

## ⚠️ 保密与合规（必读）
- **本技能适用于：作者本人投稿前自查、合作者互查、或已获作者授权的模拟评审。**
- **不得用于正式盲审（把他人未发表稿件/标书喂给 AI）**——这违反多数期刊的审稿保密协议，也违反 NIH（NOT-OD-23-149）与 NSFC 2025 年起"评审环节禁用生成式 AI"的规定。若用户是被指派的正式审稿人/评委，提醒其：不要上传受保密约束的稿件，改用本技能提供的清单自行核对。

## 先确认
- 评的是**论文稿件**还是**基金标书**？（结构与标准不同）
- 用途是自查/互查/授权评审中的哪种（涉及保密时先提示上一条）。
- 稿件在哪（用户粘贴，或放 `uploads/`；PDF 稿可先用 `fulltext-retrieval` 的 `pdf_to_md.py` 转成 Markdown 再逐节读），目标期刊/基金渠道是什么。

## 一、论文评审流程
1. **整体判断**：研究问题是否重要、设计是否合理、结论是否被数据支撑、是否适合该期刊。给一句总体印象。
2. **逐节剖析**
   - **引言**：动机与缺口清楚吗？假设明确吗？
   - **方法**：能复现吗？样本量/纳排/随机/盲法/伦理是否交代？
   - **统计**：检验是否匹配数据类型；有无多重比较校正、把握度分析；p 值与效应量/置信区间是否都报；有无 p-hacking 迹象。
   - **结果**：与方法对应吗？图表是否清晰、有无过度美化或误导性坐标轴。
   - **讨论**：结论有没有超出数据；局限是否诚实交代。
3. **报告规范核对**：按研究类型对照现行规范（不确定研究类型或规范时查 [EQUATOR Network](https://www.equator-network.org)）：

   | 研究类型 | 规范（现行版本） |
   |---|---|
   | 随机对照试验 | CONSORT（2025 更新版）；试验方案 SPIRIT（2025） |
   | 系统综述 / Meta | PRISMA 2020；诊断准确性 meta 用 PRISMA-DTA；观察性 meta 用 MOOSE |
   | 观察性研究（队列/病例对照/横断面） | STROBE |
   | 诊断准确性 | STARD 2015 |
   | 预测模型 / 临床 AI | TRIPOD；含机器学习用 **TRIPOD+AI**；影像 AI 参 CLAIM、早期临床 AI 参 DECIDE-AI；试验用 CONSORT-AI / SPIRIT-AI |
   | 动物实验 | ARRIVE 2.0 |
   | 病例报告 | CARE |
   | 质性研究 | COREQ / SRQR |
   | 卫生经济学评价 | CHEERS 2022 |
   | 质量改进 | SQUIRE 2.0 |

4. **机械/统计可核对项**（逐条查，这些最容易漏）：摘要与正文数字是否一致、表内百分比加和是否合理、有无试验注册号（ClinicalTrials.gov / ChiCTR）、伦理批号、数据可用性声明、软件及版本；统计复算存疑处可交 `data-analysis` 技能。
5. **参考文献**：可疑或像 AI 编的，交给 `reference-check` 技能核查真实性。
6. **给结论**：Accept / Minor revision / Major revision / Reject 倾向 + 理由；并按四维各 1–5 分（重要性、严谨性、新颖性、呈现质量）给分。

## 二、标书评审流程
按资助方标准打分：
- **NIH（2025-01 起简化评审框架）**：**Factor 1 Importance**（Significance + Innovation，打分）、**Factor 2 Rigor and Feasibility**（Approach，打分）、**Factor 3 Expertise and Resources**（Investigator + Environment，通常不单独打分、只评是否适配）；Overall Impact 仍按 1–9 打分。（旧的五项独立打分已废止。）
- **国自然**：意义、创新、方案可行性、研究基础；并核对**科学问题属性**是否与内容匹配；通讯评审给**综合评价 A/B/C/D + 资助意见**。

逐项指出强弱与可改进处，评"有无致命伤"。

## 输出格式
写到 `outputs/review_report.md`：
- **给编辑/评委的总评**（推荐倾向 + 总体理由 + 分维打分）
- **给作者的意见**：分 Major / Minor，每条按**四元组**写——**位置（节/页/图表）| 问题 | 依据（规范条款或文献）| 可操作改法**。
- **亮点**（也要写，公允）

**出 PDF**：报告写完交给 `render-pdf-doc` 技能渲染成 `outputs/review_report.pdf`。中文字体已自动处理，无需手动指定。

## 约定
- **两种模式**：①**投稿前自查模式**——可给出具体改写建议；②**模拟审稿人模式**——只评不改、不替作者改写结论。开工先明确是哪种。
- 意见要**具体、可执行、对事不对人**；每条尽量指到段落/图表，走上面的四元组。
- 拿不准的地方标"建议作者澄清"，不要臆断。
- 统计问题要说清"为什么是问题、该怎么做"。
