# 论文 / 标书评审技能

以严谨、建设性的审稿人视角给出评审意见。参考 K-Dense / davila7 的 peer-review 技能。**目标是帮作者改好，不是挑刺羞辱。**

## 先确认
- 评的是**论文稿件**还是**基金标书**？（结构与标准不同）
- 稿件在哪（`uploads/` 或用户粘贴），目标期刊/基金渠道是什么。

## 一、论文评审流程
1. **整体判断**：研究问题是否重要、设计是否合理、结论是否被数据支撑、是否适合该期刊。给一句总体印象。
2. **逐节剖析**
   - **引言**：动机与缺口清楚吗？假设明确吗？
   - **方法**：能复现吗？样本量/纳排/随机/盲法/伦理是否交代？
   - **统计**：检验是否匹配数据类型；有无多重比较校正、把握度分析；p 值与效应量/置信区间是否都报；有无 p-hacking 迹象。
   - **结果**：与方法对应吗？图表是否清晰、有无过度美化或误导性坐标轴。
   - **讨论**：结论有没有超出数据；局限是否诚实交代。
3. **报告规范核对**：按研究类型对照——RCT→CONSORT，系统综述→PRISMA，观察性→STROBE，动物→ARRIVE，诊断→STARD。
4. **参考文献**：可疑或像 AI 编的，交给 `reference-check` 技能核查真实性。
5. **给结论**：Major / Minor revision / Reject 倾向 + 理由。

## 二、标书评审流程
按资助方标准打分（NIH：Significance / Innovation / Approach / Investigator / Environment；国自然：意义、创新、方案可行性、研究基础）。逐项指出强弱与可改进处，评"有无致命伤"。

## 输出格式
写到 `outputs/review_report.md`：
- **给编辑/评委的总评**（是否推荐、总体理由）
- **给作者的意见**：分 Major / Minor，逐条编号、指出位置、说清问题、给出可操作建议
- **亮点**（也要写，公允）

**出 PDF（正式报告默认交付版）**：评审报告写完后，把 `outputs/review_report.md` 交给 `render-pdf-doc` 技能渲染成 `outputs/review_report.pdf`。中文报告务必指定中文字体（`--cjk-font`：本地 `Microsoft YaHei`，服务器 `Noto Sans CJK SC`），否则会漏字。

## 约定
- 意见要**具体、可执行、对事不对人**；每条尽量指到段落/图表。
- 拿不准的地方标"建议作者澄清"，不要臆断。
- 统计问题要说清"为什么是问题、该怎么做"。
- 不改写作者的研究结论；只评估与建议。
