# 基金标书技能

帮用户把课题写成有说服力、结构完整的标书。参考 franklee16 / K-Dense 的 grants 技能；预算部分参考 qinyan-academic-skills 的 `research-grants`（MIT）。**内容基于用户提供的真实课题与前期工作，不编造数据、成果或报价。**

## Python 环境（可选，用于成品排版）
> 没有项目根 `.venv`？先运行 `env-setup` 技能建好并装依赖。
```
.venv\Scripts\python.exe
```

## 先问清楚
- 申请渠道（国自然面上/青年、NIH R01/R21、省市课题…）——决定结构与篇幅。
- 课题方向、核心假设（可先用 `topic-selection`）。
- 申请人前期工作、已有数据/预实验、团队与平台条件。

## 结构模板

**国自然（NSFC）**
1. 立项依据与研究意义（国内外现状 + 亟待解决的问题 + 本项目的科学假说）——引真实文献
2. 研究目标、研究内容、拟解决的关键科学问题
3. 研究方案与技术路线（含可行性）、实验设计、可能的问题与对策
4. 特色与创新之处
5. 年度研究计划与预期成果
6. 研究基础与工作条件（前期积累、团队、平台）
7. 研究经费预算与预算说明（见下「经费预算」）

**NIH（Specific Aims + Research Strategy）**
- Specific Aims（1 页：背景缺口→长期目标→中心假设→2-3 个 Aim→影响）
- Significance / Innovation / Approach（每个 Aim：rationale、design、expected outcomes、pitfalls & alternatives）
- Budget & Budget Justification（见下「经费预算」）

## 经费预算

预算不是最后凑数字，评审会看合理性。**每一条都挂到具体研究内容/Aim，写清测算依据（单价×数量、人月×费率），别只给总额。**

**国自然（NSFC）** —— 按直接费用科目列，每项写测算依据与必要性：
- 设备费（含试制/租赁）、材料费、测试化验加工费、燃料动力费
- 差旅/会议/国际合作与交流费、出版/文献/信息传播/知识产权事务费
- 劳务费（研究生/博后/临时人员，无比例上限但需据实测算）、专家咨询费
- 间接费用按依托单位规定比例提取（有分段上限），单列说明
- ⚠️ **青基/优青/杰青等已实行经费包干制，无需编制预算明细表**——只写经费大致使用方向即可，别去编科目表。面上、重点等仍需编制预算。

**NIH（及 NSF/DOE 等海外）** ——
- Personnel：PI / Co-I / 博后 / 研究生，各写 effort（calendar months + %FTE）+ 薪资 × 比例 + fringe benefits（按机构费率）
- Equipment / Supplies / Travel / Other（发表费、动物费、测序费、分包合同…）
- 每项 justification 挂到对应 Aim，说明必要性
- R01 常用 **modular budget**（$250K/年直接费为一个模块）；R21 有经费上限，注意别超

产出：预算表 + 预算说明段落，写进 `outputs/proposal.md`（表格可用 Markdown 表）。

## 写作要点
- 逻辑链闭环：**问题 → 假说 → 目标 → 内容 → 方案** 一一对应，别断裂。
- 创新点写"与现有工作比新在哪"，不是罗列会做什么。
- 可行性用前期数据/预实验/团队条件支撑（用用户给的真实材料）。
- 关键论断引真实文献（可配合 `search-lit` 检索、`fulltext-retrieval` 下全文）。

## 产出
- 草稿写 `outputs/proposal.md`。
- **出 PDF（正式标书默认交付版）**：定稿后把 `outputs/proposal.md` 交给 `render-pdf-doc` 技能排成投稿级 `outputs/proposal.pdf`。中文标书务必指定中文字体（`--cjk-font`：本地 `Microsoft YaHei`，服务器 `Noto Sans CJK SC`），否则会漏字。

## 约定
- **绝不虚构预实验数据、发表记录、合作单位**——缺的向用户要，标"待补充"。
- **绝不虚构预算报价、单价、人员薪资/费率**——用用户提供的真实数据或标"待补充"，别自己编数字凑总额。
- 涉及人/动物实验，提醒补伦理与合规内容。
- 写完给一份自查清单（逻辑、创新、可行性、**预算合理性**、格式、字数）。
