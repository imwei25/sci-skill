# 原创研究论文写作技能（IMRaD）

把用户**真实的**研究数据、方法与结果，写成一篇逻辑闭环、可投稿的研究型论文。**绝不虚构数据、结果、样本量、统计数字或参考文献**——缺的向用户要、标"待补充"。本技能是本套件"写作本体"，负责把前面各环节的产物组织成成稿。

## Python 环境（可选，用于配套脚本/排版）
> 没有项目根 `.venv`？先运行 `env-setup` 技能。
```
.venv/Scripts/python.exe   # Windows
.venv/bin/python           # Linux / macOS
```

## 先问清楚（写之前必须对齐）
- **研究类型**：RCT / 队列 / 病例对照 / 横断面 / 诊断试验 / 动物实验 / 病例报告 / 系统综述+Meta……（决定该套哪个报告规范，见下）。
- **目标期刊**：有没有目标刊？（决定字数、结构、摘要格式、参考文献风格与上限、是否结构式摘要）。没定可先用 `research-scan` 摸行情或让用户给 2-3 个候选。
- **手上材料**：数据文件、已跑的统计结果、图表、前期文献。缺统计就先用 `data-analysis`，缺图就用 `nature-figure`，缺文献就用 `search-lit`/`literature-review`。
- **语言**：中文核心 / 英文 SCI（决定语域与引用格式；中文刊多用 GB/T 7714）。

## 全自动编排（何时自动往下、何时停下问）
**自动往下走**（确定性步骤，只汇报不打断）：从已有数据算统计→建证据/结果表→按结构成文→写完自动跑 `reference-check` 查引用→（英文/中文按需）跑 `humanize-academic` 去 AI 味→排版。
**必须停下问用户**：核心结论与主张（不能替作者拔高）、目标期刊未定、涉及未提供的数据、作者/单位/基金号等真实信息、终稿定稿前。
**失败即上浮**：某步（如检索）失败先按降级路径换道，换道再失败才问用户。

## IMRaD 写作流程

### 1. 标题 + 摘要
- **标题**：准确、含关键变量与研究类型，不夸大。
- **摘要**：按目标刊要求选结构式（Background/Methods/Results/Conclusions）或非结构式；**摘要里的每个数字必须与正文、与统计结果一致**；给关键效应量+置信区间，不只写"有显著差异"。

### 2. 引言（Introduction）——倒三角
- 背景与临床/科学意义 → 已知什么（引真实文献）→ 缺口是什么 → 本研究的问题与假设/目的。**最后一段明确研究目的**，不要写成综述。

### 3. 方法（Methods）——以"能被复现"为标准
- 研究设计、时间地点、伦理批准与知情同意（有则写批号）、注册号（试验必填，ClinicalTrials.gov/ChiCTR）。
- 对象：纳入/排除标准、抽样、分组。
- 变量与测量、干预/暴露与对照、结局定义（主要/次要）。
- **样本量/把握度依据**（缺就让用户补或用 `data-analysis` 算）。
- **统计方法**：逐一说明每个分析用什么检验、软件及版本、显著性水平、缺失值处理、多重比较校正——**方法与"结果"里出现的每个分析一一对应**（统计选择见 `data-analysis` 的护栏）。

### 4. 结果（Results）——只陈述、不解释
- 先描述样本（可配 Table 1 基线特征表）。
- 按主要→次要结局顺序报**效应量 + 95%CI + 精确 p 值 + n**，别只写 p<0.05。
- 图表：出版级图交 `nature-figure`（森林图/KM/火山图等）。**正文数字、表内数字、图中数字三处必须一致**。
- 不在结果里下结论或讨论。

### 5. 讨论（Discussion）
- 首段用一句话回答研究问题（呼应目的）。
- 主要发现的解释与机制、与既往研究的异同（引文献）、临床/科学意义。
- **局限性**诚实交代（设计、样本、偏倚、可推广性）。
- 结论**不超出数据能支撑的范围**。

### 6. 参考文献
- 每条都来自真实检索（`search-lit`/`literature-review`），**绝不凭记忆造引用**。
- 成稿后**必须**把参考文献交 `reference-check` 技能核查真实性（揪 AI 常编的假引用）。
- 中文刊多用 GB/T 7714，英文刊按目标刊风格。

### 7. 报告规范自检（按研究类型对照，投稿前必做）
RCT→CONSORT；系统综述/Meta→PRISMA 2020；观察性→STROBE；诊断准确性→STARD；预测模型/临床 AI→TRIPOD(+AI)；动物→ARRIVE 2.0；病例报告→CARE；卫生经济→CHEERS 2022。逐条核对、缺项补齐（详细清单可用 `peer-review` 技能）。

## 附属产出（按需）
- **投稿信 (cover letter)**：致编辑，1 页——研究要点、为何契合该刊、原创性与无一稿多投声明、通讯作者信息。基于真实信息，不吹。
- **审稿意见回复 (response to reviewers)**：逐条列出审稿人意见→逐条回应（做了什么修改、在稿件哪处）→附修订说明。语气专业、就事论事。
- **改投适配**：换目标刊时按新刊的字数/结构/摘要/引用格式调整。

## 产出与交付
- 稿件写 `outputs/manuscript.md`（或工作区约定路径）；cover letter 写 `outputs/cover_letter.md`；回复写 `outputs/response_to_reviewers.md`。
- **出投稿版**：Word 稿交 `render-docx` 技能（医学期刊多要 .docx）；需要 PDF 交 `render-pdf-doc`（中文字体已自动处理）。
- **去 AI 味**：定稿前把正文交 `humanize-academic`（中英文都支持），并跑其不变量校验确保数字/引用未被改动。

## 硬约束
- **绝不虚构**数据、结果、数字、样本量、统计量、参考文献、伦理批号、注册号、基金号、作者单位——缺的标"待补充"向用户要。
- 结论不超出数据；不替作者夸大主张。
- **AI 使用合规**：多数期刊要求披露 AI 辅助写作，且不接受实质由 AI 生成、无人负责的稿件。**本技能产出为草稿，作者须实质性审阅、修改并对全部内容负责**；生成时提醒用户据目标刊政策如实披露。
- 涉及人/动物研究，提醒补伦理与合规内容。
